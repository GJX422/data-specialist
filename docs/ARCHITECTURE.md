## 数据流

### 1. 数据采集层

- **云之家API**：工作日报、请假、报销、付款、采购等审批数据
- **金蝶API**：销售订单、采购入库、成本数据等
- **天气API**：气象数据（用于环境×生产联动分析）

### 2. 数据同步层

- **智能同步策略**：状态类表采用两趟查询（近30天新建 + 所有审批中/待提交）
- **产线全量同步**：7条产线每日4轮全量同步
- **增量同步**：金蝶数据每日增量同步

### 3. 数据存储层

- **MySQL数据库**：66张有效表，存储所有业务数据
- **权限分离**：
  - `finebi_ro`：只读用户（FineBI使用）
  - `yunzhijia_rw`：读写用户（同步脚本使用）

### 4. 报表生成层

- **产供销存日报**：每日自动生成，包含生产、销售、采购、库存
- **工作日报分析**：LLM驱动，7大维度分析
- **数据健康看板**：实时监控数据源状态
- **驾驶舱**：管理层关键业务指标

### 5. 报表推送层

- **IMA知识库**：所有报表推送到IMA"报表系统"文件夹
- **飞书通知**：重要报表通过飞书DM通知用户

## 核心模块

### 同步编排器 (`yunzhijia_sync_all_mysql.py`)

负责协调所有同步任务：
- 状态类表：执行两趟查询策略
- 产线表：调用独立的全量同步脚本
- 金蝶表：增量同步

### 状态类同步脚本

支持以下参数：
- `--start/--end`：指定日期范围
- `--status`：按状态过滤（RUNNING,RETURNED）
- `--all`：全量同步（仅产线表支持）

### 报表生成脚本

- `gen_daily_report.py`：产供销存日报
- `gen_health_dashboard.py`：数据健康看板
- `gen_gm_cockpit.py`：驾驶舱
